# Design Critic — 프로젝트 개요 문서
> Claude Code 작업 지시서 겸 프로젝트 명세서  
> 최초 작성: 2026

## 1. 프로젝트 한 줄 요약

CI(Corporate Identity) 이미지와 설명 텍스트를 업로드하면,  
**마시모 비넬리의 디자인 원칙**을 기반으로 해당 디자인을 평가하고 개선 방향을 제시하는 웹앱.

---

## 2. 프로젝트 배경 및 목적

### 배경
- 한국의 지자체 로고는 단체장이 바뀔 때마다 교체되는 경우가 많고, 결과물의 완성도가 낮은 경우가 빈번함
- 반면 일본 지자체 로고는 전국시대 다이묘 깃발(家紋) 전통에서 비롯된 설계 원칙 — **전쟁터에서 순간 식별 가능해야 한다** — 이 내재되어 있어 식별성이 뛰어남
- 영국, 프랑스 등 서유럽 지자체 로고도 역사적 문장(Heraldry) 전통을 계승하여 일관성과 내구성이 높음
- 이 차이를 관통하는 원칙은 하나: **기능이 형태를 결정한다(Form follows function)**

### 핵심 통찰
좋은 디자인에는 트렌드와 무관하게 작동하는 원칙이 있다.  
그 원칙을 **마시모 비넬리(Massimo Vignelli)의 디자인 철학**에서 가져온다.

### 목적
- 동네 간판집, 소규모 디자이너가 **무료로** 자신의 디자인을 평가받을 수 있는 도구 제공
- AI가 직접 디자인을 대신 해주는 것이 아니라, **스스로 개선할 수 있는 기준과 피드백**을 제공
- 고기집 간판 하나라도 더 나은 디자인이 될 수 있도록

---

## 3. 평가 철학 — 비넬리 원칙을 "체(Sieve)"로 사용

### 왜 비넬리인가
마시모 비넬리는 뉴욕 지하철 노선도, 아메리칸 에어라인 CI 등을 설계한 20세기 대표 그래픽 디자이너.  
그의 원칙은 스타일이 아닌 **구조와 기능**에 관한 것이어서 시대를 초월해 적용 가능하다.

### 비넬리 핵심 원칙 (PDF에서 추출 예정)
Claude Code 작업 시 `vignelli_canon.pdf`를 읽어 아래 항목을 보완할 것.

현재 파악된 주요 원칙:
1. **시각적 힘 (Visual Power)** — 한눈에 인식되는 강렬함이 있는가
2. **지적 우아함 (Intellectual Elegance)** — 불필요한 장식 없이 의도가 명확한가
3. **시간을 초월하는 내구성 (Timelessness)** — 유행이 지나도 촌스럽지 않을 것인가
4. **적절성 (Appropriateness)** — 업종·맥락에 맞는 형태와 색인가
5. **일관성 (Consistency)** — 크기와 매체가 달라져도 동일하게 작동하는가

### 평가의 전제
- 정답은 없다. 그러나 원칙은 있다.
- 트렌드(플랫 디자인, 뉴모피즘 등)는 참고하되 평가의 주축이 아님
- 비넬리 원칙이 **1차 필터(체)**, 업종 적합성이 **2차 필터**

## 4. 기능 명세

### 4-1. 입력
| 입력 항목 | 필수 여부 | 설명 |
|---|---|---|
| CI 이미지 | 필수 | JPG, PNG, WebP. 간판 사진, CI 파일 모두 허용 |
| 업종/카테고리 | 필수 | 예: 정육식당, 미용실, 지자체, IT스타트업 |
| 디자인 의도 설명 | 선택 | 색상의 의미, 형태의 의미 등 제작자가 담으려 한 내용 |
| 타겟 고객 | 선택 | 예: 중장년층 동네 주민, 20대 직장인 |

### 4-2. 출력 — 평가 리포트
다음 6개 항목에 대한 서술형 피드백 + 각 항목별 간단한 개선 제안:

1. **식별성** — 멀리서, 빠르게, 처음 보는 사람도 인식되는가
2. **단순성** — 불필요한 요소(그라데이션, 과도한 폰트 수, 장식)가 있는가
3. **의미 일치성** — 제작자가 텍스트로 설명한 의도와 형태·색이 실제로 부합하는가
4. **내구성** — 3~5년 후에도 시대에 뒤처지지 않을 디자인인가
5. **적용 가능성** — 간판 대형 출력, 명함, 앱 아이콘, 흑백 인쇄 등 다양한 환경에서 작동하는가
6. **업종 적합성** — 해당 업종이 주는 신뢰감, 전문성, 분위기와 일치하는가

> 점수(숫자)는 부여하지 않는다. 숫자는 오해를 낳는다.  
> 대신 각 항목에 **"잘 되고 있는 점 / 개선이 필요한 점 / 구체적 제안"** 3단 구조로 서술.

### 4-3. 언어
- 입력 텍스트의 언어를 감지하여 동일 언어로 피드백
- 기본값: 한국어

## 5. 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React (Vite) | 브라우저 실행, 빠른 프로토타이핑 |
| AI | Anthropic Claude API (`claude-opus-4-8`) | 이미지 + 텍스트 동시 처리 가능한 멀티모달 |
| 스타일링 | Tailwind CSS | 빠른 UI 구성 |
| PDF 처리 | Claude Code에서 직접 읽기 | 비넬리 PDF → 시스템 프롬프트 원칙으로 변환 |
| 배포 | 정적 호스팅 (Vercel 또는 GitHub Pages) | 무료, 간단 |

## 6. 시스템 프롬프트 설계 방향

Claude Code 작업 시 구현할 AI 에이전트의 시스템 프롬프트 핵심 구조:

```
당신은 마시모 비넬리의 디자인 원칙을 체득한 디자인 평론가입니다.

[비넬리 원칙 — vignelli_canon.pdf에서 추출한 내용 삽입]

평가 원칙:
- 좋은 디자인의 적이 무엇인지 알고 있습니다: 불필요한 복잡성, 유행 추종, 맥락 무시
- 당신은 디자인을 대신 만들어주지 않습니다
- 당신은 제작자가 스스로 개선할 수 있도록 구체적인 방향을 제시합니다
- 비판은 명확하게, 그러나 제작자의 의도를 존중하며 서술합니다
- 동네 간판집 사장님도 이해할 수 있는 언어로 설명합니다

입력:
- 이미지: [CI 이미지]
- 업종: [업종]
- 제작 의도: [텍스트]
- 타겟: [타겟]

출력 형식:
[식별성 / 단순성 / 의미 일치성 / 내구성 / 적용 가능성 / 업종 적합성]
각 항목: 잘 되고 있는 점 → 개선 필요한 점 → 구체적 제안
```

## 7. Claude Code 작업 지시

### Step 1 — PDF 읽기 및 원칙 추출
```
vignelli_canon.pdf를 읽어서:
1. 비넬리가 명시한 디자인 원칙 목록을 추출
2. 각 원칙의 핵심 문장을 한국어로 요약
3. 시스템 프롬프트에 삽입할 형태로 정리
결과를 vignelli_principles.md로 저장
```

### Step 2 — React 웹앱 구현
```
위 명세에 따라 React + Vite + Tailwind 웹앱 구현:
- 이미지 업로드 UI
- 텍스트 입력 폼 (업종, 의도, 타겟)
- Claude API 호출 (multimodal)
- 평가 리포트 렌더링
```

### Step 3 — 시스템 프롬프트 통합
```
vignelli_principles.md의 내용을 시스템 프롬프트에 통합
Claude API 호출 시 system 파라미터로 전달
```

### Step 4 — 테스트
```
테스트 이미지 3종으로 평가 품질 검증:
1. 동네 고깃집 간판 (열악한 디자인 예시)
2. 잘 된 지자체 로고 (일본 사례)
3. 스타트업 CI
```

## 8. 비고 — 철학적 메모

이 앱은 "더 예쁜 디자인"을 만드는 도구가 아니다.
